Port Scanning like a Pro

Port Scanning like a Pro

I'm writing about this because it seems to me that many people overlook this crucial step. There's also a lot of bad information about it on the Internet. So let's get to it.

Port scanning is the first step when approaching a target IP. The IP might've come from many sources like resolving domains & subdomains or scanning IP ranges for active hosts.

So, with an IP address we just need to run a portscanner tool such as nmap, but there's a mistake that's very common to make when running those tools: Not testing all ports. You should test all ports.

You should test all ports!

When fuzzing other things such as subdomains, subdirectories, we're never sure when to stop, there's always more to test, a better wordlist. But with portscanning is different, there are only (2^16) ports (65635), so why do we say to only test a fraction of those?

I've heard people say:

"It's hidden, when I run nmap it doesn't even show up!"

It's not hidden, that just happens because nmap doesn't test all ports by default. I think you always should do so, we spend a lot of time running all sorts of tools, why are we so hasty when it comes to port scanning?

A practical example

I always check all the ports, and right now I'm using naabu + nmap with the command naabu -host HOST -p - -nmap-cli 'nmap -sV -oX nmap.xml'. naabu will check for all the open ports and then invoke nmap to enumerate the running services in the found ports. The -sV option provided to nmap is very important, because it'll show us which kind and version of service is running in each port, documentation here.

PORT     STATE SERVICE VERSION
8002/tcp open  http    nginx 1.22.0

This is how the nmap version & service enumeration output looks like.

The command above will spit out a nmap.xml file with that juicy data in xml format. I find XML hard to work with, so I convert it to JSON using something like xq.

Some interesting idea would be to have a cronjob to run nmap once a day and diff the results. There's a lot more that you can do with port scanning, specially with a powerful tool like nmap and some good scripts. But as it is, this is a good starting point.
